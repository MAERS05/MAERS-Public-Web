# MAERS - 项目更新日志

## [2026-03-09]

### 💅 界面 (UI)

- **沉浸式背景**：为各个模块卡片增添了沉浸式背景图功能，并优化了全屏展示自适应与平滑切换体验。

### 🔧 底层 (Core)

- **环境配置脚本**：新增 `install_deps.bat` 与 `install_libraries.py`，用于一键自动安装项目所需的 Python 与 Node.js 依赖库。

## [2026-03-07]

### 🚀 功能 (Feat)

- **Videos 模块正式上线**：
  - 新增 `videos.html`，采用与文学/游戏一致的网格布局渲染影视内容。
  - **海报比例适配**：卡片采用 2:3 竖屏比例（140px * 210px），完美适配影视封面。
  - **Music & Videos 联动**：Videos 页面标题支持交互，可一键跳转回 Music 模块。
  - **Admin 深度集成**：后端入口更名为“影视管理”，支持封面上传、标签管理及 MD 详情编辑。

### 💅 界面 (UI)

- **卡片标签系统优化**：
  - **强制单行截断**：全模块（文学、影视、游戏、记录）卡片标签实现超长文本自动省略（Ellipsis），彻底解决长标签溢出卡片边界的问题。
  - **空间利用率提升**：通过精细化缩减标签间距 (Gap) 与内边距 (Padding)，在保持美观的同时显著增加了单行标签的文字显示容量。
  - **布局稳定性**：引入 `box-sizing: border-box` 与容器宽度锁定机制，确保在各种极端文本长度下卡片布局均不发生形变。

### 🔄 重构 (Refactor)

- **CMS 渲染引擎增强**：
  - `cms-render-nav.module.js`：`renderPageTitle` 现在支持 HTML 标签渲染，允许标题内嵌交互式 `span` 和 `onclick` 事件。

### 🔧 底层 (Core)

- `_studio/clean-data.py`：新增 `_videos` 附件目录与 `videocovers` 封面目录的清理保护。

## [2026-03-05]

### 🚀 功能 (Feat)

- **Games Playing 视图**：新增随机抽取游戏功能。
  - 进入游戏页面默认显示 Playing 视图（点击标题处 `Playing/Played` 标签切换）。
  - Playing 视图中央展示一个游戏封面卡片，点击"随机抽取"按钮后等待 2 秒显示随机结果。
  - 抽取结果的封面卡片行为与 Played 网格完全一致：hover 显示标题/标签，点击进入 md 文档，存入最近访问栏。
  - 抽取结果通过 `localStorage` 持久化（key: `maers_playing_game_id`），下次进入页面自动恢复上次结果。
  - 使用 `cms-render-grid.module.js` 的 `createGridItem`（已对外导出）渲染卡片保证 100% 行为一致性。

### 🔄 重构 (Refactor)

- **Games Admin 并入 CMS Admin 统一架构**：
  - 删除独立的 `admin-games.html`，游戏后台改由 `admin-cms.html?module=games` 统一入口访问。
  - `admin-portal.json` 更新游戏管理入口 URL 并补充 `style` 字段。
  - `admin-main.module.js` 新增 `GamesView` 导入，按 `CURRENT_MODULE` 自动分发 `CustomView`（`literature` → `LiteratureView`，`games` → `GamesView`，其余 → `null`）。
  - `games/main.module.js` 加入 `!State.IS_ADMIN` 保护，确保视图切换按钮仅在前台渲染。

### 🔧 底层 (Core)

- `cms-render-grid.module.js`：将 `createGridItem` 改为 `export function`，供外部模块复用。
