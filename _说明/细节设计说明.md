## 1. 极速防闪烁机制 (Flash Guard)
**位置**: `index.html` (Head & Body)

### 问题背景
在深色模式下，浏览器渲染页面时往往会先展示默认的白色背景，然后再加载 CSS 变黑。这会导致看到一瞬间刺眼的“白屏闪烁” (FOUC - Flash of Unstyled Content)，尤其是在夜间访问时体验极差。

### 解决方案
我实施了一套只有 **0.15秒** 存在感，但从不缺席的“三层防御”机制：

1.  **第一层 (HTML Style)**: 
    *   直接在 `<html>` 标签内联 `style="background-color: #0a0a0a"`。
    *   **作用**: 在 CSS 文件完全加载前，抢先告诉浏览器“背景是黑的”，物理阻断第一帧白屏。
2.  **第二层 (Critical CSS)**: 
    *   在 `<head>` 中内联定义 `#flash-guard` 样式。
    *   **优化**: 我舍弃了复杂的纹理图片，只使用纯色 `#141417`。这不仅将 `index.html` 体积减小了 50% 以上，而且通过后期动画的配合，完全感知不到起初缺少纹理。
3.  **第三层 (Guard Element)**: 
    *   一个物理存在的 `<div id="flash-guard">` 覆盖全屏。
    *   **动作**: 页面加载完成后，它不会立即消失，而是执行 `0.15s` 的淡出动画。
    *   **目的**: 这 0.15s 的缓冲足以掩盖页面内部组件（如图片、字体）加载时的微小跳动，给呈现“一打开就是完美状态”的错觉。

### 为什么是 0.15s？
*   **0.1s**: 太快，偶尔会露出底下的加载过程。
*   **0.3s+**: 会感觉到有一个“遮罩层”的存在，产生阻滞感。
*   **0.15s**: 这是一个**感知阈值**。感觉网页是“瞬间打开”的，但视觉神经又刚好被平滑的过渡安抚，不会觉得突兀。

## 2. 标题导航彩蛋 (Title Navigation)
**位置**: `index.html` (.brand-title)

### 交互细节
首页顶部的巨大标题 "MAERS SPACE" 不仅仅是装饰，它实际上是两个独立的隐藏文字链接。
*   点击 **"MAERS"** -> 跳转至 **个人主页 (me.html)**
*   点击 **"SPACE"** -> 跳转至 **空间概览 (space.html)**

### 设计考量
我不希望在首页放置显眼的 "关于我" 或 "进入空间" 按钮，破坏极简的视觉平衡。将被动展示的标题转化为可交互的导航入口，既保持了界面的整洁，又为探索者提供了一个惊喜的发现路径。

## 3. 移动端性能特化 (Mobile Optimization)
**位置**: `style.css` (@media max-width: 1024px)

### 问题与取舍
在桌面端，我使用了大量的 `backdrop-filter: blur()` (毛玻璃) 和复杂的 SVG 遮罩动画。但在手机上，这些是电池杀手和掉帧之源。为了极致的流畅度，我必须做“减法”。

### 针对性优化
1.  **移除毛玻璃**: 手机端直接禁用 `backdrop-filter`。改为使用高不透明度的纯色背景 (95% 黑 / 98% 白)。
    *   *结果*: 滚动 FPS 从 40 提升至稳定 60。
2.  **简化切换动画**: 手机端的日夜切换不再运行复杂的 `clip-path` 几何计算，而是降级为最稳健的 `opacity` 淡入淡出。
3.  **隐藏星空动画**: 屏幕太小，星星不仅看不清，还徒增 GPU 负载，因此直接 `display: none`。

## 4. "上帝视角"收缩模式 (Shrink View)
**位置**: `style.css` (html.shrink-view)

### 交互创新
当按住 `Ctrl` 滚动或点击特定UI时，页面并不只是传统的缩放。
*   同时增加圆角和深重阴影，让当前的网页变成一张“悬浮在桌面上的卡派”。
*   背景的星空纹理此时会完全暴露出来，给予一种特别的体验。

## 5. 后台统一开场动画 (Admin Unified Splash)
**位置**: `admin*.html`, `static-style/splash.css`

### 设计意图
后台管理系统相比前台展示页，加载速度慢，为了掩盖各个模块初始化时的 DOM 变动与资源加载并防止闪烁，我引入了一个极致简约但统一的开场动画。

### 视觉语言
*   **统一符号**: 全系后台页面统一使用 **"🛠️" (Hammer and Wrench Emoji)** 作为加载图标。这不仅代表了“构建与维护”的含义，而且 Emoji 原生矢量、无须网络请求的特性，保证了这一图标永远不会“加载失败”。
*   **黑色帷幕**: 与前台的防闪烁机制一脉相承，使用 `<div id="splash-screen">` 承载纯黑背景。

### 动效参数 (The "Snappy" Tuning)
经过反复调试，我将动画参数锁定在了**“干脆利落”**的区间，以符合管理员追求效率的心理：
1.  **Delay (0.2s)**: 仅给予浏览器 200ms 的喘息时间来完成首帧渲染。
2.  **Transition (0.5s)**: 遮罩淡出的时间严格控制在 0.5s。这个时长既不会像 0.3s 那样生硬（让人觉得像是丢帧），也不会像 0.8s 那样拖泥带水。
